# v1.18.0 - 2026-04-25

Release "hardening + ergonomie" : feature flags, scripts d'orchestration, audit securite complet, durcissement CI.

## Highlights

- **Feature flags ON/OFF** via `srv-docker.env` - premier toggle `WAZUH_ENABLED`. Backend retire le blueprint, frontend cache l'entree menu et bloque la page. Helper PHP `feature_enabled()` reutilisable pour d'autres modules. Voir [[02_Domaines/feature-flags]].
- **Scripts d'orchestration** : [[Code/start.sh]] (env-merge auto + lancement), [[Code/stop.sh]] (down avec garde-fou `-v`), [[Code/maj.sh]] (pipeline 5 etapes git pull -> env-merge -> build -> migrations -> up -d). Voir [[02_Domaines/scripts-orchestration]].
- **`scripts/env-merge.sh`** : merge des cles manquantes de `srv-docker.env.example` vers `srv-docker.env` SANS toucher aux valeurs existantes. Backup auto. Idempotent. Appele par start.sh + maj.sh.
- **Audit securite complet** : 0 vulnerabilite exploitable trouvee, 6 fixes defense-in-depth appliques (CSRF api_proxy.php, pin SHA actions GitHub, bandit/pip-audit bloquants PR, test-server refuse fallback passwords, docker-compose.prod.yml retire bind-mounts).
- **CI Semgrep** (`p/owasp-top-ten` cross-langue PHP+JS+Python+Docker), bloquant PR + main.
- **E2E [[Code/tests/e2e/go-security.mjs]]** codifie 8 invariants : POST sans CSRF -> 403, GET non-auth -> redirect, XSS escape DOM, hash chain integrity. Si un nouvel endpoint contourne ces controles, le test casse.

## Migration

- [[08_DB/migrations/043_ssh_audit_schedules_machines_target]] etend l'ENUM `target_type` pour 'machines' (multi-select v1.17 portait sur le scheduler mais le schema bloquait).

## Changements de workflow

- **Ancien** : `docker compose up -d`, `git pull && docker compose build && up -d`, `cp srv-docker.env.example srv-docker.env` puis copier-coller a la main les nouvelles cles.
- **Nouveau** : `./start.sh -d` (avec sync auto de l'env), `./maj.sh` (pipeline complet).

## Rappels operationnels

- Wazuh OFF en prod : `WAZUH_ENABLED=false` dans srv-docker.env puis `./maj.sh` ou `docker compose up -d`.
- Apres un commit upstream : `./maj.sh`.
- Pour stopper proprement : `./stop.sh` (et `./stop.sh -v` SEULEMENT si on veut wiper la BDD).
