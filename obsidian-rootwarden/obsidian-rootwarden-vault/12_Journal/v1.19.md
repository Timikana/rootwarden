# v1.19.0 - 2026-04-29

Release "audit acces SSH" : inventaire detaille + suppression chirurgicale des cles SSH presentes sur les serveurs distants.

## Highlights

- **Inventaire detaille** des `authorized_keys` par user distant. Compteur cliquable dans `/adm/server_users.php` -> modal listant chaque cle avec type, fingerprint SHA256, comment, badge proprietaire (cross-ref `users.ssh_key`), date 1re vue. Migration [[08_DB/migrations/044_server_user_ssh_keys]].
- **Scan via root** : 1 seul `execute_as_root` qui dump tous les authorized_keys du serveur (root + users a home protege inclus). Avant : N appels SSH en simple user, silent fail sur `/root/*`.
- **Suppression chirurgicale d'une cle** : bouton ✗ par cle dans le modal. SSH en root, recalcule fingerprint via `ssh-keygen -lf`, filtre la ligne ciblee, conserve le reste. Garde-fou anti-self-lock : suppression de la cle plateforme RootWarden BLOQUEE par defaut (override via `force=true`).
- **Drift detection gratuite** : cles disparues entre 2 scans -> DELETE auto. Combine avec `first_seen_at`, on peut alerter quand une cle inconnue apparait.
- **3 fixes prod v1.18 deployment** : `chmod env-merge`, fallback `ip:port` pour `test_platform_key`, install Wazuh multi-OS (apt/yum-dnf/zypper).
- **Bit +x persistant en git** (mode 100755) sur `maj.sh`, `start.sh`, `stop.sh`, `scripts/*.sh`, `scripts/sync-obsidian-vault.py`. Plus de chmod manuel apres `git pull`.

## Cas d'usage cle

- **Audit "qui a acces"** : pour chaque user distant, vois en 1 clic toutes les cles deposees, leur proprietaire (si user RW connu) et celles "fantomes".
- **Nettoyage post-incident** : tu retires UNE cle suspecte sans casser les autres.
- **Compliance ISO27001 / SOC2** : preuve d'inventaire d'acces SSH avec horodatage 1re vue.
